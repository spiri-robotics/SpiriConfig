# Development

## Running locally

```console
$ uv sync
$ uv run esc-config
```

Or in the container, the way the store runs it, with live reload:

```console
$ docker compose -f compose.yaml -f compose.dev.yaml up --build
```

There is also a `.devcontainer/`, so VS Code or GitHub Codespaces can open this
repository directly inside the build image ("Reopen in Container").

## Tests, linting, docs

```console
$ uv run pytest
$ uv run ruff check .
$ uv run sphinx-build -b html docs docs/_build/html
```

The `Makefile` wraps these as `make test`, `make lint`, and `make docs`.

## Releasing

`scripts/release.py` is the single writer of the version. It edits
`pyproject.toml` and `compose.yaml` together, updates `uv.lock`, commits, and
tags:

```console
$ uv run scripts/release.py patch     # or minor, major, or an explicit X.Y.Z
$ git push && git push --tags
```

Pushing the tag triggers `.github/workflows/publish.yml`, which builds the image
and pushes it to `ghcr.io/spiri/esc-config` at the new
version and at `latest`.

## Updating from the template

This project was generated by a Copier template. To pull in later improvements to
the template's best practices:

```console
$ copier update
```
